Tech knowledge part 5 (here, i'll talk a bit about my problem solving process)

Got blocked on a simple problem

What was the problem?

I was getting errors with an exercise on freeCodeCamp called "Profile Lookup". As the title suggests, it's meant to look up certain information saved in a 'javascript object'. The problem was that it was meant to warn that a particular contact didn't exist if it couldn't find the name that was given, but it was always saying that it couldn't find data even when a particular person did have information in the object.

What problem solving techniques did you use?

I started off using 'console.log()' to see what the program was doing. That helped me look through the code and formulate some ideas for solutions. I had some pseudocode to help plan as well. After spending too much time on the problem with no progress, I did a google search and found a direct solution on 'stackOverflow'.

How did you feel throughout the process?

Very frustrated. I couldn't wrap my head around how I could fix the bug and I was spending so much time on it.

What did you learn?

Probably one of the most important and enlightening things about programming. Conditional statements let us control how, when and/or if things happen. But more importantly, 'control flow', which statements the program executes and in what order is also very important in determining the when-if-how a statement executes.

Solved a problem in an elegant way

What was the problem?

It was an exercise on freeCodeCamp called "Record Collection". The problem was that the program needed to return a set of date based on some search information given to it and it would throw an error when it couldn't find an existing record of a property that was given.

What problem solving techniques did you use?

Reading the error being thrown and using 'console.log()' was the reason I found out what the problem was, in the first place. I had been struggling for a while at this point, so I reached out to Amandine. I wrote out some pseudocode to organise my plan to catch the problem. I tried out my new code in isolation (as in, I turned off other behaviours so I could watch the effects of this new statement). I tweaked code a few more times when problems came up and finally got to a solution with just one or two more statements, each responding based on the input given.

How did you feel throughout the process?

It was very encouraging once I got Amandine's help. She basically dropped the solution in my lap. XD It was great to get over a block I had with someone's help.

What did you learn?

In this case, some more detail about how we access information from 'javascript objects'. Also, it was an opportunity to look very hard at how much time I let pass before asking help to find a solution.

Reflect on how confident you feel using the problem solving techniques and process

I'm pretty confident with all the problem solving techniques. Of course, console.logging and error messages are only as helpful as the context dictates, but other than that; I love the fact that in the modern day we live in, we have access to essentially infinite information through the internet. And naturally, being a gamer, I love trying things out, especially from different angles and new ways. As for reflection, I'm sure there's still room for improvement there, (as well as with the other techniques) but I'm doing alright. I'm often making sure that, especially after a rough block, I'm looking back at the situation and thinking about how quickly I went for help and how much I explored the issue myself and what I did to finally solve the problem.
