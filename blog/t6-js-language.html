In the JavaScript language, describe all the use cases for each of:

parentheses ()

Parens (parentheses) are used for so many things. They're math operators used to force an equation to prioritise solving one part first. 5 * (2 + 1) = 15 They're used to enclose arguments to a block of code. function (argument) {do stuff} Here, the code you put in between parens tells the function what code to reference when it performs its task. They're also good for grouping different data types together if ( ((1 + 1) + 'turtle doves') ) {'It's the second day of xmas}

brackets []

These are nice. They let you declare arrays and then access a value from both arrays and objects. With arrays it's pretty straightforward [3] gets the value in the fourth 'box' in the array (the count starts from 0, remember?) And here's a lengthier explanation for objects. An object stores not just a numbered box like the array. It has a property/value pair 

          var object = {
          key1: value1,
          key 2: value2
          }
In this case object.key1 gives us value1. But to get value2 we need object[key 2] since we're checking a property that had a space. It's a similar case for checking values based on a pre-defined variable or an argument.

braces {}

One word. Blocks. You must've already noticed that i've been using these braces. They just say; all code inside us shall henceforth belong to the preceding function, if statement or loop.

single quotes '' AND double quotes ""

These are essentially interchangeable markers for strings. It mainly becomes a question of 'what do I want to display?' Whatever quotes happen to be outside will mark it's contents as a string. So 'Mary's little lamb' would be confusing to your computer. It wouldn't know whether to end the string after Mary's quote or the quote after lamb. Double quotes here would keep the poor computer from getting a headache. :3
